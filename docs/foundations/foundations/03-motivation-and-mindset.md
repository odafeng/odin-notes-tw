# 動機與心態

> 改寫自 The Odin Project：[Motivation and Mindset](https://www.theodinproject.com/lessons/foundations-motivation-and-mindset)
> ｜Foundations › Introduction

## 核心概念

學寫程式很有成就感，但同時也會讓人受挫。它跟任何值得學的技能一樣需要時間累積，不是一個週末、甚至不是一個月就能學會的。只要你願意投入時間與心力，任何人都能學會寫程式。在真正進入課程主體之前，這一課先談談能幫助你成功的心態，以及你該避開的陷阱。

### 抓住你的動機

先花點時間想清楚：你為什麼決定學寫程式？可能是想要一份有成就感又待遇好的職涯、被寫程式帶來的創作自由吸引、想擁有做出任何 app 的能力，或是想把某個點子變成自己的公司。你的動機可能是上面幾項的組合，也可能完全是別的原因。不論是什麼，都要牢牢抓住它，因為在這條路上撐不下去的時刻，正是這個動機會把你拉到終點，給你一個明確的目標。

### 學習的過程：focus mode 與 diffuse mode

學會一個概念之後動手練習，能讓你更完整理解事物是怎麼運作、怎麼拼在一起的。而 project（專案）是驗證「理論理解」與「實際運作」是否吻合的終極方法。

學習時，你的大腦會不斷在兩種模式之間切換：

- **focus mode（專注模式）**：你有意識地專注在學習、閱讀、看影片或寫專案的時候。
- **diffuse mode（發散模式）**：發生在你沒有主動學習的潛意識裡，例如洗碗、運動或睡覺時。這時大腦會把你剛學到的東西跟既有的知識連結起來，很多「突然想通」的突破就發生在這個階段。

了解這兩種狀態很重要，因為你可以善用它讓學習更有效率。當你卡在一個概念或專案上時，先認真嘗試解決問題，然後去休息一下、讓潛意識繼續運作，往往回來就找到答案了。關鍵順序是：**先努力想，再去休息**。

一句話總結學習方法：理解它、練習它、然後教會別人。把你會的東西教給別人，是鞏固所學最好的方式，也常會暴露出你原本沒察覺的知識漏洞。你可以透過在社群裡幫助別人來練習這個方法。

### 卡住的時候怎麼辦

你一定會在某個地方卡住，可能是某個概念難以理解，也可能是專案裡有東西就是不動。遇到時可以用這三招脫困：

- **Google 它**：幾乎可以確定，世界上一定有人遇過跟你一樣的問題，一次搜尋往往就能找到解法。
- **休息一下**：讓 diffuse mode 幫你在背景處理問題。
- **求助前先做功課**：在社群（例如 TOP 的 Discord 伺服器）發問時，帶著你已經研究過的內容一起問。當別人看到你已經自己努力過，會更願意幫你。

### 關於 AI 生成程式碼

近年 LLM（大型語言模型）與生成式 AI 在程式補全（如 GitHub Copilot）與程式生成（如 ChatGPT）上爆紅。這些工具很強大，但對初學者來說，過度依賴會在你的知識裡留下盲點與缺口：

- 你會錯過「搞懂某件事怎麼運作、為什麼這樣做」的機會。
- 問出好問題本身是一項技能，凡事問 AI 而不問人，會拖慢這項能力的養成。
- 談論自己寫的程式碼也是一項職場技能（例如 code review 時要解釋你的做法與理由），靠社群發問能順帶練到它。
- AI 的輸出必須仔細審視，而初學者通常還沒有足夠的判斷力去分辨好壞。
- 寫好 prompt 固然是技能，但它是「附加」的，撐不起基礎程式能力；而 Odin 要教的正是這個基礎。
- AI 被設計來「給你答案」，而不是幫你培養研究與解決問題的能力。你問人，對方可能反問你的理解、引導你自己找到解法。
- 缺少研究、解決問題與批判思考的練習，面試會很吃力，因為多數面試不允許用 AI。

因此，TOP 不建議你在學習階段使用 AI 工具。

### 管理你的學習時間

**持續**投入比一週衝一次更有效。養成每天在固定時間、帶著明確目標學習的習慣，才能穩定進步。

每個人掌握概念的速度不同，這無關聰明與否，只是各自的人生經驗不同罷了。TOP 沒有先修條件，也沒有截止日期，是完全自訂步調的：你可以花足夠的時間把一課學紮實再往前。自己硬訂 deadline 只會帶來不必要的壓力，逼你趕進度、囫圇吞棗。這門課很吃研究能力，你無法保證每次都能剛好快速找到對的文章，但過程中你會學到大量日後受用的東西。放輕鬆，享受這趟旅程——**別擔心，去學就對了**。

## 常見陷阱

!!! warning "把時間耗在這些坑裡，動能就沒了"
    以下是初學者最常踩的雷，請盡量避開：

    - **拖延（Procrastination）**：最大的敵人。用 Pomodoro Technique（番茄鐘工作法）對治：專注工作 25 分鐘、休息 5 分鐘，完成四輪後休息 15–30 分鐘。25 分鐘不長，不容易讓人找藉口拖延。
    - **只盯著最終成果**：整天想著「學完之後要幹嘛」「還要多久」，焦慮到動彈不得——這其實是另一種拖延。改成專注在「過程」：給自己訂一週要完成幾個番茄鐘的目標，從小量開始慢慢加。
    - **完全不休息**：長時間連續苦讀會導致 burnout（倦怠），反而降低產出。專注力像肌肉，用久了需要休息才能恢復。規律地讓大腦與身體充電，你反而做得更多。
    - **數位干擾**：Email、社群通知、殺時間的網站會打斷專注。學習時關掉通知，用網站封鎖工具限制自己。
    - **實體干擾**：背景的電視聲、他人交談同樣傷害專注。找個安靜的角落，或戴上降噪耳機，圖書館也是好選擇。
    - **鑽牛角尖（Rabbit holes）**：課程連結了大量優質資源，很容易一頭栽進一個你還用不到的主題。課程的順序是刻意安排的——盡量照著路徑走，別讓岔路毀了你的節奏。
    - **跟別人比較**：跟走得更前面的人比，只會換來沮喪。只跟「過去的自己」比：這週、這個月、今年的你，是不是比以前更強了？
    - **反效果的抄筆記**：抄一堆筆記很花時間又常白費工。與其抄成「參考資料」，不如記成「日後研究的提示」，並習慣直接讀官方文件——那本來就是別人幫你寫好的筆記。
