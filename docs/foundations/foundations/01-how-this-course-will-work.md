# 這門課怎麼進行

> 改寫自 The Odin Project：[How This Course Will Work](https://www.theodinproject.com/lessons/foundations-how-this-course-will-work)
> ｜Foundations › Introduction

## 核心概念

The Odin Project（以下簡稱 TOP）是一個開源（open-source）的學習社群，目標是把你從零帶到能獨當一面的全端（full-stack）開發者。這一課不教語法、不寫程式，而是先幫你搞懂「這門課到底怎麼運作」，以及「你該用什麼心態走完它」。把這件事先想清楚，後面每一課都會更順。

### 課程如何運作

TOP 的做法是**匯整**（aggregate）網路上針對某個主題最好的內容來教你。每一課通常會先由 TOP 介紹主題、給你一些背景脈絡，然後把你導向別人做的優質外部資源（文件、文章、影片等）。也因此，多數課程結尾會有幾個「你應該答得出來」的問題（knowledge check），有些課還會附上練習（exercise）。此外，整份課程沿途安排了不少專案（project），讓你透過**實際動手做**來鞏固理解。

（在《Odin Notes TW》這份講義裡，我們把每一課消化改寫成自足的繁體中文內容，讓你不點外部連結也能讀懂重點；但原文列出的練習與專案，仍建議你回到 TOP 依步驟完成。）

### 用「滾雪球」的心態，而不是「考試」的心態

不要把 TOP 或程式學習想成學校裡的一門課。它不是那種「一次把教材塞進腦袋、然後考一次、非過即當」的東西。比較貼切的比喻是**滾雪球**：你就是那顆雪球，從積滿雪的山坡往下滾，滾得越遠，黏上的雪就越多。當然，過程中也會有雪掉下來，你會常常忘記東西，但這完全是學習的一部分。

所以，當你走到某個專案，覺得「我好像什麼都沒記住」時，別慌——**這很正常，每個人都會這樣**。專案不是在考你背了多少，而是一個個練習點：當你開始一題一題解決問題，靠 Google 搜尋和 Odin 社群求助時，該回來的知識就會回來。

### 為什麼這門課「不教」某個語言或工具

常有人問：為什麼這門課沒教某某語言、某某工具？

關鍵在於 TOP 的真正目標**不是**教你一項項孤立的技能，也不是要當一份 tutorial（教學步驟手冊）。它想教的是「為什麼（why）」，想改變你思考問題的方式。程式設計的核心從來不只是「寫 code」，而是**解決問題、問對問題、做足夠的研究**。

當你打好這個底子，往後要擴充技能就會容易得多。完成這門課後，你應該能**直接跳進任何工具的官方文件（documentation）**開始學——因為讀文件才是掌握任何工具最好的方式。很多人會被文件裡的術語（jargon）嚇到，這可以理解，因為寫文件的人通常預設你已有一定的程式背景；而這正是 TOP 給你大量材料自己閱讀、鍛鍊這項能力的原因。

一句話：如果你讀完這門課，還覺得「要學 Python 就得再找一門一模一樣的課」，那要嘛是你不相信自己，要嘛是你沒把這門課真正重要的觀念帶走。你在這裡學到的是**可遷移（transferable）的技能**——一旦你知道某件事「是可能做到的」，剩下的「怎麼做」多半就只是一次 Google 搜尋的距離。

### 關於語言（英文）

TOP 吸引了世界各地想成為開發者的人，但要先有心理準備：這份課程以英文撰寫，維護者也是英語使用者，不會（也無法）為你翻譯。你即將進入的這個世界，是**深深扎根於英文**的——程式語言的語法、教你怎麼用的文件、以及社群裡的大多數人，預設都用英文和你溝通。

如果英文不是你的母語，這個事實不是要打擊你，而是要幫你做好準備。TOP 強烈建議：對於一時看不懂的主題和術語，多花點時間；也鼓勵你另外找用母語講解同一主題的資源來加深理解，或搭配一本自己語言的翻譯字典隨查隨用。（而《Odin Notes TW》這份繁體中文講義，正是為了降低這道語言門檻而存在。）

### 接下來是什麼、以及「不要跳過任何東西」

Foundations（基礎）這一單元刻意涵蓋非常廣的主題：從網際網路與你自己的電腦開始，接著建立開發環境、學 Git 與 GitHub，再進到前端（front-end）的 HTML、CSS、JavaScript，最後淺嚐一下後端（back-end）的基本概念。走完之後，你不只會理解網路如何運作，還能分辨自己會用到的各種技術，並能做出一個有基本樣式與少許互動的簡單網頁，同時在命令列（command line）上操作自如。

這份課程由專業人士維護，資源經過嚴選與編排；找不到好資源時，他們就自己寫。因此請記住：課程裡的**每一樣東西都是刻意安排、且對你成為成功的程式設計師至關重要的**。課程是層層堆疊的——每一部分都建立在前面所有內容之上，跳過東西會在你的知識裡留下「不理解的空洞」，之後就會開始影響你解決問題、看懂任務的能力。

唯一算「選讀」的，只有明確標示為「額外資源（additional resources）」的部分，那是留給你想更深入某個主題時用的。除此之外——

!!! note "TOP 的原話"
    **不要跳過任何東西（DO NOT SKIP ANYTHING!）**

## 程式碼範例

本課以觀念與心態為主，沒有程式碼範例。真正要你動手打的第一批指令，會從後面「認識電腦／建立開發環境」等課程開始。這裡先給一個之後會反覆用到的操作示意：當你卡關時，最實際的兩個動作是「查文件」與「用關鍵字搜尋」。例如你想知道某個指令怎麼用：

```bash
# 查一個指令的官方手冊（manual）
man ls

# 或直接查指令自帶的說明
ls --help
```

先建立「遇到不懂就去查」的反射，比背下任何一條指令都重要。

## 常見陷阱

!!! warning "把它當成一次性的考試"
    最常見的心態誤區，就是期待自己「讀完一課就永久記住」。TOP 的設計是滾雪球式的累積——忘記、回頭查、再想起來，本來就是流程的一部分。專案不是背誦測驗，卡住很正常，別因此覺得自己不適合寫程式。

!!! warning "為了求快而跳課或非線性學習"
    課程層層相依，跳過內容會留下「不理解的空洞」，日後解題時才會發現地基缺了一塊，反而更慢。除非某段被明確標為額外資源（additional resources），否則不要跳過。

!!! warning "以為之後還需要另一門一樣的課"
    這門課的目的不是餵你孤立技能，而是教你「why」與自學能力。若你讀完仍覺得學新語言非得再找一門手把手的課不可，代表核心觀念還沒被你帶走——你其實已經具備直接讀文件上手的能力。
