# 選擇你的下一步

> 改寫自 The Odin Project：[Choose Your Path Forward](https://www.theodinproject.com/lessons/foundations-choose-your-path-forward)
> ｜Foundations › Conclusion

## 核心概念

先花一點時間回頭看看：走到這裡，你已經接觸了至少三種語言（HTML、CSS、JavaScript）的基礎，如果臨時被要求，你其實已經能自己拼湊出一個網頁了。

現在的你應該可以：在 command line（命令列）輸入指令而不慌張、理解「在瀏覽器輸入網址後到底發生了什麼、頁面又是怎麼回到你眼前的」、寫出一個基本網頁並用 JavaScript 讓它做一些互動、以及對這些技術如何拼成「一名 web 應用開發者的未來」有清楚的輪廓。

Foundations 只是起跑。接下來你要在兩條主線裡選一條走下去：

- **Ruby with Ruby on Rails**
- **JavaScript with Node / Express**

兩條路都會帶你累積技能、建立作品集（portfolio），最終成為一名優秀的開發者。這裡沒有標準答案。學多種語言能讓你更全面，專精一種則能讓你在特定語言上更深入——但兩條路都能把你帶到終點。**最重要的一件事是：不要走到一半又跳去另一條。** 半途換路只會讓你倒退。

### 學程式，遠不只是學一種語言

真正在學的，是「如何研究並解決困難的問題」、如何建立工程師思維、如何在複雜的程式碼庫（codebase）裡工作、如何和其他開發者有效溝通——這些能力遠比語法本身重要。而且好消息是：這些能力可以被帶著走。大多數語言在概念與結構上都很相似，因此當你學會第一個語言後，跨到第二個語言會比第一次輕鬆得多。

有人會用「你所在地區都用某某語言」來說服你選哪一條——這聽起來很有道理，但因為一個「好」的開發者能輕鬆轉移技能，你第一個學的語言並不會決定你能不能找到工作。如果這份焦慮真的揮之不去，那就去做任何能讓自己安心的事：例如查一查你所在地在地職缺用什麼技術（也別忘了遠端職缺）。但請記住——**一門好的課程，遠比你先學哪個語言重要。**

### 兩條路各自的特色

**Ruby and Ruby on Rails**

- **Opinionated（有既定慣例）**：專案結構不用你自己傷腦筋決定，這在接手別人專案時特別有幫助；你會學到「The Ruby Way」，這種寫乾淨程式的習慣也能帶到其他語言。
- **Elegant（優雅）**：提供大量工具讓程式易讀，很多解法比其他語言更精簡。
- **Rapid development（開發快速）**：Rails 讓你很快就能做出應用，相同時間內能完成更多有趣的專案。
- **Single paradigm（單一範式）**：物件導向（OOP，object-oriented programming）就是 Ruby 的做事方式，而許多其他語言也以 OOP 為主。

**JavaScript - Node & Express**

- **Un-opinionated（無既定慣例）**：沒有標準做法，專案結構得自己決定——有好有壞。
- **Very fast（很快）**：JavaScript 使用 JIT（Just-In-Time，即時編譯），運算速度很快。
- **Verbose（明確、話多）**：沒有「魔法」，你得理解每一步才能讓程式運作，這會讓你更貼近「電腦實際上是怎麼讀你的程式」。
- **前後端同一種語言**：整個應用維持同一種語言有助於寫出一致的程式，也能玩一些酷東西——例如同一份程式碼可以同時跑在瀏覽器與伺服器上做最佳化。

說到底，你選哪一條都不要緊。真正重要的是**堅持下去**，並在整個職涯裡始終保持「學習者的心態」。

## 程式碼範例

本課以觀念與心態為主，無程式碼範例。你只需要做一個決定：在 Ruby on Rails 與 Node/Express 之間，挑一條路，然後開始走。

## 常見陷阱

!!! warning "半途換跑道"
    最傷的不是「選錯」，而是「選了又反悔」。兩條路都能把你帶到終點，唯獨走到一半跳去另一條會讓你倒退——因為你會不斷重置對某條路已經累積的熟悉度與進度。先選一條，堅持走完，之後想學另一種語言隨時都可以回頭。

!!! warning "把『先學哪個語言』當成職涯生死題"
    第一個語言不會決定你能不能找到工作。決定成敗的是你解決問題、閱讀陌生程式碼、與人協作的能力——這些都能跨語言帶著走。與其為選語言焦慮，不如把力氣花在「把一門好課程扎實走完」。

## 練習

1. 回顧一下你在 Foundations 學到的東西：command line、HTML／CSS、JavaScript、Git、以及「網址輸入後發生了什麼」。確認自己對每一塊都有基本的踏實感。
2. 在 **Ruby on Rails** 與 **Node/Express** 之間做出選擇。把上面兩份特色清單當作依據，想清楚哪一種做事風格更吸引你。
3. 如果真的難以抉擇，可以用原文提供的「命運轉盤」（Path Wheel，Ruby on Rails vs Node）隨機幫你敲定——選項落在哪就走哪條，反正兩條都通。
4. 做出決定後，別回頭猶豫，直接進入你選定那條路的下一門課程。
