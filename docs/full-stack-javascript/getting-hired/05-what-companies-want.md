# 公司要什麼

> 改寫自 The Odin Project：[What Companies Want](https://www.theodinproject.com/lessons/node-path-getting-hired-what-companies-want)
> ｜Full Stack JavaScript › Getting Hired › Preparing for Your Job Search

## 核心概念

在前面的課程，你已經整理過自己的狀況：為什麼想要這份工作、為什麼覺得自己值得。但殘酷的事實是，光靠「我需要」或「我應該得到」這份工作，並不能讓你被錄取。要被錄取，你得換一個視角思考：**站在公司的立場**，他們為什麼想僱用你？他們因此會在你身上尋找什麼？

這一課的重點，就是把這個視角講清楚：招募主管（hiring manager，負責決定要不要錄取你的人）到底在找什麼，以及 internship（實習）為什麼是一條聰明的切入路徑。

### 布幕後的真實：招募主管的處境

先建立一個關鍵的同理心：招募主管是很忙的人。

想想他為什麼要開這個職缺。正是因為現有團隊的工作負荷太重、想要有人來分擔，讓團隊運作得更好、更快，他才需要「多一個人」。但諷刺的是，招募這件事本身就是額外的負擔——他每週可能要處理數百封不合格的應徵信，這些都在吃掉他寶貴的時間。他真正想做的，是趕快回去把產品做出來，而且最好一次找對人，短期內不必再重新招募。

理解了這個處境，你就能推導出他真正在乎的兩件事：他要找一個能**盡快帶來價值（add value as soon as possible）**、而且能**隨著組織一起成長（grow with the organization）**的人。

### 成長曲線：把自己呈現成一條上升線

先談「成長」這一點，這對你來說其實不難證明。

如果你能一路自學走到這裡、靠自己學會 web development（網頁開發），這件事本身就是最好的證據。關鍵在於：把自己呈現成一條**成長曲線（growth curve）**，而不是一條靜止的水平線。也就是說，你要讓對方相信——你會持續、而且快速地學習與進步，不會停下來。

你有真實的職涯目標，而且到目前為止，你已經用飛快的速度學會了很多東西（對吧？）。把這個「學得快、還在加速」的軌跡展現出來，就是在賣「成長潛力」。

### 立即帶來價值：先有雞還是先有蛋的難題

比較棘手的是「立即帶來價值」這一點。這裡對方想確認的，是你**過去有沒有相關經驗**、以及你能不能達到他們的技術門檻（technical bar）。

求職有一個惱人的 catch-22（兩難悖論）：證明你「能被僱用」最簡單的方法，就是你「曾經被僱用過」。有過工作經驗，等於在你名字旁邊打了一個勾。

要破解這個先有雞還是先有蛋的難題，你能做的就是：**盡一切辦法證明「別人曾經足夠信任你，願意採用你的作品或技能」**。這不一定要是正式的全職工作——接過的案子、參與的開源專案、被真實使用者用過的作品、幫社群或非營利組織做過的東西，都能當作「有人相信過你」的證據。

### 第三個因素：契合度（Fit）

除了上面兩個特質，還有一個常被低估、卻往往決定成敗的因素——**fit（契合度）**。

招募主管要找的，是他和他的團隊能一起愉快共事的人。前面提到的能力與動機，其實只是「被納入考慮」的最低門檻；真正讓你脫穎而出、或是讓你出局的，常常是「這個人是不是團隊成員會想更認識、想每天一起工作的人」。

這也是為什麼幾乎所有招募流程，在後期都會請整個團隊來評估候選人。技術面試過了不代表結束，團隊面試（culture fit）同樣關鍵。

### 三個關鍵字：能力、動機、契合

把上面的內容濃縮起來，招募主管通常在找三件事：

1. **Capability（能力）**：你能不能達到技術門檻、做出東西、盡快帶來價值。
2. **Motivation（動機）**：你是不是一條持續上升的成長曲線，有真實的目標與快速學習的軌跡。
3. **Fit（契合）**：團隊是不是願意、也樂意每天和你一起工作。

面試前把自己的故事對照這三點檢查一遍，你就會知道履歷、作品集、面試回答該強調什麼。

### Internships：破解經驗難題的捷徑

要開始累積經驗、向別人證明「你可被僱用」，一個很好的起點就是去爭取 internship（實習）。

這裡有一個實用的提醒：**盡量找有給薪的實習**。原因很現實——人們不會珍惜免費得到的東西。你會希望一起共事的人重視你的貢獻、覺得自己也有責任幫你成功；有給薪的關係，才會讓雙方都認真投入。

實習的好處在於：它通常更看重你的**成長潛力**，而不是你過去做過什麼。這正好幫你繞開前面那個「先有雞還是先有蛋」的難題——你不必先有經驗，才能開始累積經驗。

實習的招募流程通常和全職職缺類似，但沒有那麼嚴格。可以把實習想成是公司對你進行的一場「超長版面試」，用來評估要不要把你轉成全職。反過來看，這也是你評估這家公司的機會。

## 程式碼範例

本課以觀念為主，沒有需要執行的程式碼。你可以把三個關鍵字當成一份自我檢查清單，在準備每一次應徵前問自己：

```text
面試前自我檢查（Capability / Motivation / Fit）

[ ] Capability：我有沒有具體作品／專案能證明技術能力？
[ ] Capability：有沒有「別人用過我的作品或技能」的證據？
[ ] Motivation：我能不能講出一條清楚、加速中的學習軌跡？
[ ] Motivation：我有沒有明確的職涯目標，並說明為何是這家公司？
[ ] Fit：我了解這個團隊的文化與做事方式嗎？
[ ] Fit：我能不能讓對方覺得「想每天和我共事」？
```

## 常見陷阱

!!! warning "只從「我需要這份工作」出發"
    履歷與面試若只強調自己有多需要、多想要這份工作，其實打動不了招募主管。他關心的是「你能為團隊解決什麼」。永遠先換位到公司立場，再組織你的說法。

!!! warning "把自己講成一條靜止的水平線"
    只列出「我現在會什麼」而不呈現成長軌跡，會讓人覺得你的天花板就在這裡。要展現的是「成長曲線」——你學得快、還在加速、不會停。

!!! warning "為了破解經驗難題而接無償實習"
    免費付出的東西，別人往往不會珍惜。盡量選有給薪的實習，讓雙方都認真投入、也讓你的貢獻真正被重視。

!!! warning "以為技術過關就穩了"
    能力只是最低門檻。fit（契合度）常常才是決勝點，而且後期通常整個團隊都會參與評估。別忽略「讓人想與你共事」這件事。

## 練習

1. 花點時間從招募主管的視角理解招募這件事。不用鑽牛角尖，目標是弄清楚「他們為什麼想僱用你」。可以搜尋並閱讀幾篇關於「如何招募軟體工程師」的文章與部落格，特別留意招募方描述的痛點與需求。其中一個經典觀點來自 Joel Spolsky：好的工程師要同時滿足兩點——**1. 聰明（Smart）**，以及 **2. 能把事情做完（Gets things done）**。閱讀時，把看到的重點對回本課的三個關鍵字：Capability、Motivation、Fit。

2. 進一步了解 internship（實習）。搜尋「如何取得軟體實習（how to get a software internship）」相關的討論與經驗分享，看看別人是怎麼找到、以及如何準備實習的。同時可以用求職平台（例如 WayUp、LinkedIn、實習專屬招募網站）實際瀏覽目前開放的實習機會，感受一下市場上在找什麼樣的人。

3. 讀完後，用本課的自我檢查清單，替一份你真的有興趣的職缺（或實習）寫下你在 Capability、Motivation、Fit 三方面各自的「證據」與「還缺什麼」。這會直接成為你之後寫履歷與準備面試的素材。
