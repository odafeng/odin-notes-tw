# 建立專業人脈

> 改寫自 The Odin Project：[Professional Networking](https://www.theodinproject.com/lessons/node-path-getting-hired-professional-networking)
> ｜Full Stack JavaScript › Getting Hired › Preparing for Your Job Search

## 核心概念

在成為 web developer（網頁開發者）的路上，一個穩固的專業人脈（professional network）是無法被取代的資產，而且它的價值會延續到你職涯的每一個階段。人天生就是會與他人連結、接受幫助、也給出幫助的社會性動物；在軟體圈也不例外，無數的社群每天都在進行這種互利的交換，這正是 networking（人脈經營）的本質。

專業人脈最實際的一個好處，是它有機會幫你找到軟體業的第一份工作。這不是保證，但確實是許多開發者的親身經歷。凡事都是一分耕耘一分收穫：你在人脈上投入愈多，它回報你的機率就愈高。但要與人建立真誠的連結需要時間，所以我們強烈建議你「愈早開始愈好」，不要等到要找工作了才臨時抱佛腳。

### 什麼是專業人脈？

先釐清一個常見的誤解：你的專業人脈「不是」你在社群媒體上追蹤的那些人，也不一定等於你的朋友。它指的是那些你已經建立起真誠連結的人，不論是線下面對面，還是線上互動——你們之所以連結，是因為有共同的興趣或關注點，而且彼此連結能創造出比單打獨鬥更多的價值。這些人是你可以開口求助的對象，而你日後也能回過頭來幫助他們。

對「成為軟體開發者」這條路最有幫助的人脈，主要由軟體開發從業者，以及相關角色（例如 recruiter（招募人員）或管理職）組成。現在就盤點一下自己在這個領域的人脈吧——很有可能，你會發現它還有很大的擴展空間。

### 為什麼專業人脈有用？

回想一下，你一定有過卡在某個任務、然後開口向人求助的經驗，這就是在「動用你的人脈」。在軟體開發的世界裡，你同樣可以用這套方式來蒐集資訊、做決定，以及找工作。

**蒐集資訊**：搜尋引擎和 AI 都很好用，但當你需要更細膩、更有脈絡的答案時，直接問一個真人往往更快。舉幾個例子：

- 你想找一位資深開發者做 informational interview（資訊型訪談），了解他們的日常工作，以及當初是怎麼進入這一行的。
- 你想請人給你的 portfolio（作品集）一些回饋。
- 你在尋求職涯發展的建議。

跟一位真正有相關經驗的人互動，好處在於你可以形成雙向對話、追問後續問題、並得到即時回饋——這是冷冰冰的搜尋結果給不了的。

**做決定**：你有沒有經歷過「decision paralysis（決策癱瘓）」？也就是選項多到反而選不出來。這在科技圈特別常見，尤其對新手來說，語言、framework（框架）、工具多到眼花撩亂。這時候，向你信任的人徵詢意見、參考他們的推薦，往往輕鬆許多。這麼做其實是在蒐集「social proof（社會證明）」：當認識的人推薦某樣東西，我們就更傾向選它——這也是為什麼你在網購前會先看評價。

**找工作**：談到求職，人脈很可能是你手上最有價值的資產。根據不同來源的統計，多達 80% 的職缺其實從未公開張貼在網路上。換句話說，當你在瀏覽 job board（求職網站）時，你看到的大約只有 20% 的職缺，剩下的都藏在所謂的「hidden jobs market（隱藏職缺市場）」裡。這個比例會因來源而異，但可以肯定的是：隱藏職缺市場真實存在，而且規模不小。如果你的求職策略完全沒有試著打進這個市場，等於是自斷後路，只能跟一大票人去搶那少數公開的職缺。

站在公司的角度想想就懂了：招募其實很痛苦。在理想世界裡，每當有新職缺，一位優秀的候選人就會立刻自己出現，公司就能省下張貼職缺、篩選履歷、批改技術測驗、進行多輪面試的整套流程。這些全都很花時間、也很燒錢。所以完全可以理解，為什麼多數公司更偏好透過隱藏職缺市場來招人。

那要怎麼打進隱藏職缺市場？答案就是「透過你的人脈」。當你的人脈把你視為一個值得信任、具備特定技能的人，你可以確信在有合適機會出現時，他們會想到你。你可能在四處打聽工作時，就先一步收到某家公司即將開缺的風聲；更好的情況是，有人直接把你推薦給某個職位。許多公司甚至會發獎金給成功內推候選人的員工，所以「經營人脈、幫忙傳名字」對雙方都有誘因。

這並不是說你就該完全放棄投遞公開職缺——那些當然還是值得爭取，但重點在於「重質不重量」：鎖定那些你真正感興趣、核心價值觀跟你契合的公司。與其用散彈打鳥的方式投出幾百份一模一樣的履歷，不如寄出更少、但為每家公司量身打造的申請，成效反而更好。

### 如何建立專業人脈

就像鍛鍊技術能力一樣，經營人脈也需要用心與策略。**在地人脈（local network）**讓你建立起紮實的面對面連結，**虛擬人脈（virtual network）**則讓你能和世界各地的人往來。兩者都有價值，而同時兼顧兩者的策略最為穩固。

人脈需要時間累積，愈早開始愈好。現在就開始建立連結，代表你日後真的需要倚靠人脈時，手上會有更多可以動用的資源。可以考慮主動把學習 web 開發的一部分時間，撥出來做這件事：它可以是你在高強度寫程式之間的一個喘息，而且在參加 local meetup（在地聚會）或研討會時，你甚至能把「經營人脈」和「學習」結合在一起。

**Informational interview（資訊型訪談）** 是快速取得可靠資訊的絕佳方法。它的正式定義，就是和一位「身處你嚮往的職位、或你想深入了解的領域」的人聊聊。你可以問對方一些問題，例如：「你是走哪條路走到現在這個位置的？」「你典型的一個工作天大概在做什麼？」「對像我這樣的人，你會給什麼建議？」「談談你在這個角色裡遇到的挑戰吧。」這些問題應該幫助你判斷某個職位或某家公司適不適合你，以及你的策略是否需要調整。這同時也是與對方建立連結的好方法——訪談之後，明智的做法是把對方加進你的人脈（例如在 LinkedIn 上連結）。

### 建立真誠的連結

專業人脈的「經濟法則」獎勵的是真誠與相互尊重。如果你要開口請人幫忙，前提是你已經先建立了真誠的連結，這樣得到幫助的機率會高得多。反過來說，請千萬不要「冷不防（cold）」地找上一個陌生人，開口就要跟對方連結，或直接提出「幫我進你公司拿到這份工作」「請當我的 mentor（導師）」這類要求。這可能會被視為不尊重。大家都很忙，會把有限的時間留給他們認識且尊重的人。

比較好的策略，是帶著對「這個人」本身的真誠興趣去展開對話，並在共同的興趣點上建立連結。舉例來說，你可以先看看對方在 LinkedIn 上的個人檔案，找出你們的共通點；或在線下用「你做什麼工作？」「你是怎麼走到現在的？」「日常工作裡遇到哪些挑戰？」這類問題開啟對話。重點是「讓話題圍繞著對方，而不是你自己」。展現出你對這個人本身感興趣，而不是想從他身上拿到什麼。認真傾聽對方說的話、追問後續問題，如果你有能力，甚至可以主動提出幫忙。把這種互動想成「約會」：你不會在第一次見面就要求對方當你的伴侶，對吧？人脈也是同樣道理。如果第一次對話進行得順利，之後再提出連結的請求也不遲。

如果你不擅長在現場跟陌生人攀談，而你正要去參加在地聚會或研討會，那就事先準備一些話題。想好幾個通用問題，例如「你在哪裡工作？」「你們公司用什麼 tech stack（技術棧）？」「你當初怎麼進入寫程式這一行的？」另外，別忘了也要準備「介紹自己」：想好一段 30 秒的 elevator pitch（電梯簡報），說清楚你是誰、你在做什麼、以及你希望 12 個月後走到哪裡。這能讓你安心不少，因為當腦袋在當下一片空白時，你至少有話可說。

如果是線上而你還不知道怎麼起步，那就找一兩個「看起來很擅長跟他人建立連結」的人，觀察他們的做法：他們多久發一次貼文、多常在別人貼文底下留言？他們用什麼樣的語氣？他們怎麼對待與自己互動的人？很有可能，當你把其中一些做法內化成自己的風格後，你在連結他人這件事上也會更順利。

### 維護你的人脈

專業人脈是一個動態、有機的存在。它就像一座花園，需要一些維護與照料才能保持良好狀態。你可以透過在線上社群裡發問與回答問題、傳訊息給你的連結、參加線下活動，或定期和最親近的連結約 coffee chat（咖啡閒聊）來維護它。理想上，這一切不該讓你覺得像在完成苦差事，而是你因為享受、也真的從互動中獲得價值，而自然而然去做的事。

關於建立專業人脈，還有太多可以談，但這一課足以讓你有個好的開始。記住：它需要時間，所以趁早開始、保持真誠。祝你好運！

## 程式碼範例

本課以觀念為主，沒有程式碼。不過你可以把「準備素材」當成一個可以動手完成的小任務，先寫下屬於你自己的 elevator pitch 與訪談問題清單，隨身帶著備用：

```text
# 我的 30 秒 elevator pitch（電梯簡報）
- 我是誰：正在轉職的自學 web developer，專注在 JavaScript / Node.js
- 我在做什麼：目前在打磨 portfolio，剛完成一個全端專案（放上 GitHub 連結）
- 12 個月後的目標：成為一名初階全端工程師，加入重視程式碼品質的團隊

# Informational interview 常備問題
1. 你是走哪條路走到現在這個位置的？
2. 你典型的一個工作天大概在做什麼？
3. 對像我這樣正在入行的人，你會給什麼建議？
4. 你在這個角色裡遇到最大的挑戰是什麼？

# 現場破冰問題
- 你在哪裡工作？
- 你們公司用什麼 tech stack？
- 你當初怎麼進入寫程式這一行的？
```

## 常見陷阱

!!! warning "不要「冷開場」就伸手要好處"
    最容易毀掉一段潛在連結的行為，就是完全沒有互動基礎，一上來就要求對方「幫我進你公司」「當我的 mentor」，或直接丟出連結請求。這很容易被解讀成不尊重對方的時間。正確順序是「先建立真誠連結、後才提出請求」：先讓話題圍繞著對方、認真傾聽、找到共同點，等第一次互動順利之後，再自然地提出下一步。把它想成約會——沒有人會在第一次見面就要求對方定下來。

!!! warning "別把人脈當成臨時求救按鈕"
    許多人只在「要找工作了」才想到經營人脈，但真誠的連結需要時間累積，臨時抱佛腳往往來不及。人脈應該是你平時就在澆灌的花園，而不是求職時才臨時打開的水龍頭。趁早開始、定期維護，等你真正需要時它才派得上用場。

!!! warning "只盯著公開職缺，等於自願走窄路"
    如果你只在 job board 上投履歷，你面對的其實只是整個就業市場約 20% 的職缺，卻要跟最多的競爭者去搶。這不代表你該完全放棄公開職缺，而是要「重質不重量」——鎖定你真正感興趣的公司、寄出量身打造的申請，同時透過人脈去接觸那 80% 的 hidden jobs market。

## 練習

你的時間有限、選項卻近乎無窮，所以請先了解自己手上有哪些選項，然後挑一兩個開始行動。

1. **在地（線下）人脈**：機會會依你所在的地區而不同。大城市通常有成熟的科技活動與社群，小地方則可能較有限。可以從以下方向盤點你身邊的機會：
    - 上 Meetup、LinkedIn 或用 Google 搜尋，找出聚焦某項技術的在地社群。
    - 翻閱在地商業刊物，留意本地研討會與科技活動的公告。
    - 問問親朋好友，看看有沒有可以介紹給你認識的軟體開發者。
    - 在社群媒體上追蹤在地科技公司，掌握他們舉辦的線下活動。
    - 訂閱聚焦本地科技圈的電子報。
    - 自己發起一個 Meetup 社團，主題可以是 Ruby、JavaScript 等你有興趣的技術。

2. **虛擬（線上）人脈**：選項更廣，平台與社群多不勝數。以下幾個方向可以幫你決定要把心力放在哪：
    - 加入 The Odin Project 的官方社群（Discord），這是最貼近本課程學員的起點。
    - **LinkedIn**：你的個人檔案等同於一份線上履歷，第一印象很重要，值得花時間好好優化。接著追蹤你想任職或覺得有趣的公司，以及你能學習的個人，用有品質的提問和留言去互動，也可以考慮開始創作關於自己開發旅程的內容。當有人與你互動時，主動連結，並透過瀏覽對方檔案、找出共同點來讓連結更真誠。
    - **Discord**：幾乎每種語言或框架都有專屬的 Discord 頻道，你可以在裡面發問、求助。註冊後搜尋你感興趣的主題（例如 Node.js、Ruby on Rails），再申請加入。
    - **開源專案（Open Source）**：當你具備足夠技能與信心去貢獻開源專案時，你會直接和專案 maintainer（維護者）與其他貢獻者協作。這是一邊為共同目標努力、一邊認識其他開發者的絕佳方式。
