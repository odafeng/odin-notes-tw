# 篩選職缺

> 改寫自 The Odin Project：[Qualifying Job Leads](https://www.theodinproject.com/lessons/node-path-getting-hired-qualifying-job-leads)
> ｜Full Stack JavaScript › Getting Hired › Applying to and Interviewing for Jobs

## 核心概念

當你開始找工作，很快就會蒐集到一長串職缺 lead（線索／潛在機會）：招募網站上的職位、朋友轉介的公司、你主動關注的團隊。問題是，你的時間與精力有限，而每一份申請、每一次面試準備、每一輪來回溝通都要花掉數小時甚至數天。如果對每一個 lead 都投入相同的心力，你會很快耗盡體力，卻把大把時間浪費在「就算錄取了你也不會去」的職缺上。**篩選職缺（qualifying leads）** 的目的，就是在投入申請成本之前，先用一套自己的標準把清單排序，讓最值得你花時間的機會浮上來。

### 從你已經定好的標準開始

在前面的步驟裡，你應該已經把「評估一份工作的標準」想清楚，並且記錄在一份試算表（spreadsheet）裡。這份試算表就是篩選的基礎。標準通常分成兩類：

- **必備條件（must-have）**：沒有就直接淘汰的底線。例如薪資不能低於某個數字、必須能遠端、必須在你能通勤或願意搬遷的地點、必須用你能接受的技術棧。
- **加分條件（nice-to-have）**：有了更好、沒有也還能接受的項目。例如團隊氣氛、學習資源、產品你是否感興趣。

篩選的第一刀很簡單：逐一檢視清單上的每個 lead，**把不符合任何「必備條件」的直接劃掉**。這一步會立刻縮短你的清單，也讓你不再對那些注定不會接受的職缺抱有幻想。

劃掉不合格的之後，剩下的職缺再依「加分條件」排序。你不需要弄一套複雜的評分系統——**越簡單越好**。給每個加分條件一個權重，替每份工作打分、加總，就能排出一個偏好順序。不要為了追求精確而把系統做得過於龐大，那反而會讓你遲遲無法開始行動。

### 用「期望值」排序：把價值乘上錄取機率

單看「這份工作有多好」還不夠，因為再理想的職缺，如果你幾乎不可能拿到，投入大量心力也是浪費。The Odin Project 建議在試算表裡多加一個欄位：**用合理努力拿到這份工作的機率（百分比）**。

接著把這個機率乘上「這份工作對你的價值」（也就是上一步用加分條件算出的分數），得到一個新的欄位——可以稱之為 **expected value（期望值）**。用期望值來排序，你就得到一份「哪些工作最值得你花時間」的清單。

這個做法背後的邏輯是：

- 一份非常理想但你錄取機率極低的工作，期望值可能不高；
- 一份條件普通但你很有把握拿到的工作，期望值反而可能更高；
- 期望值排在前面的，才是你應該優先、認真投入的對象。

這不代表你完全不能碰高風險、低機率的夢幻職缺。你仍然可以送出一些「練習用（practice）」的申請——把它們當成磨練履歷與面試的機會。差別在於，經過篩選之後，**你很清楚哪些公司值得你花力氣去投**，而不是盲目地對每一家都全力以赴。

### 為什麼一定要先篩選

篩選最重要的價值，是防止你**被流程吸進去而不自覺**。求職流程一旦啟動，很容易讓人陷進去：你花了時間寫申請、對方回信了、約了面試、你又花時間準備……不知不覺投入了大量心力，最後才發現這根本是一份你不會接受的工作。這種「沉沒成本陷阱」出現得比你想像中更頻繁、更容易。

先做篩選，等於在投入之前就替自己踩一次煞車，逼自己回答：「就算拿到 offer，我真的會去嗎？」把答案是「不會」的職缺提前排除，你省下的時間可以集中到真正重要的機會上。

### 加分條件可以再想得更細

除了薪資與地點這類明顯的項目，下面這些「加分條件」也值得放進你的評估表一起考量。它們往往決定了你進去之後過得好不好、成長得快不快：

- **在這份工作能不能持續學習？** 對剛入行的人來說，成長速度往往比一開始的頭銜或薪水更重要。
- **有沒有 mentor（導師）與教學資源？** 有人願意帶你、公司有沒有指導文化，會大幅影響你的進步。
- **同事的水準如何？** 你能從身邊的 peer（同儕）身上學到多少？強的團隊會把你一起拉高。
- **每天去上班的心情如何？** 你會期待進辦公室，還是感到抗拒？這關乎長期的可持續性。
- **這份工作的 job security（工作保障）風險有多高？** 別以為大公司就穩如泰山——大企業一樣會經歷重組（restructuring），也常常沒什麼好理由就裁員。
- **薪資是否足以支撐你的生活？**
- **地點是否符合你與家人的需求？**
- **你打算做這份（第一份）工作多久？** 對第一份工作的期待，會影響你願意在其他條件上妥協多少。

把這些問題化成欄位、給定權重，你的篩選表就會更貼近真實的決策，而不只是比薪水數字。

### 特別小心「培訓＋派遣」型的機會

在篩選過程中，有一類職缺要格外警覺：某些公司（The Odin Project 點名 Revature、FDM 這類）會提供「先付費培訓、再把你派去客戶端當顧問（consultant）」的方案。這類機會表面上像是入行捷徑，實際上藏著不少陷阱，值得你在劃掉「必備條件」之前特別檢查：

- **培訓期時薪極低**（文中舉例約每小時 8–15 美元），而且你往往無法自由選擇要學哪種技術。
- **培訓期沒有合約保障、淘汰率高**，你可能在最脆弱的階段被無預警刷掉，毫無申訴空間。
- **綁約條款嚴苛**：常見是兩年合約，提前解約要付高額違約金（文中舉例 1 萬至 4 萬 5 千美元不等）。更糟的是，培訓期與等待派遣的空窗期常常**不計入合約年限**，實際上可能把你綁到約兩年半。
- **薪資明顯低於市場行情**：文中舉例年薪約 4.5 萬至 5.5 萬美元，而同級入門職位的平均約為 7.6 萬美元；薪水也常不隨當地生活成本調整。
- **待派期間（bench）的薪資接近最低工資。**
- **搬遷要求嚴苛**：可能被要求短時間內多次搬家，搬遷補助卻不足以覆蓋實際開銷，而且你無法選擇被派去哪裡，還可能被卡在難以解除的租約裡。

評估這類（其實也適用於所有）機會時，可以這樣做：**上 Glassdoor、Reddit 這類平台查公司評價；仔細讀完每一條合約條款，特別注意遇到人生變故時的保障；在簽約前算清楚自己總共要承擔多少財務義務；並確認自己有足夠存款撐過培訓、搬遷與待派的空窗期。**

### 小結

篩選職缺的整個流程可以濃縮成幾個動作：用已定好的標準劃掉不合格的 lead → 依加分條件排序 → 乘上錄取機率算出期望值 → 依期望值決定投入順序 → 對高風險機會的合約與口碑保持警覺。核心精神只有一句話：**把有限的時間，花在真正值得的機會上。**

## 程式碼範例

本課以求職觀念為主，沒有需要執行的程式碼。不過「篩選」本身就是一套可以用試算表或簡單資料結構表達的計算邏輯。下面用一段最小的 JavaScript 示範「期望值排序」的概念，幫助你把腦中的評估流程具體化：

```javascript
// 每個 lead：value 是依加分條件算出的價值分數，
// odds 是「用合理努力拿到這份工作」的機率（0～1）。
const leads = [
  { company: "A 公司", value: 90, odds: 0.2 }, // 很理想但很難拿到
  { company: "B 公司", value: 60, odds: 0.8 }, // 條件普通但很有把握
  { company: "C 公司", value: 75, odds: 0.5 },
];

// 期望值 = 價值 × 錄取機率
const ranked = leads
  .map((lead) => ({ ...lead, expectedValue: lead.value * lead.odds }))
  .sort((a, b) => b.expectedValue - a.expectedValue);

for (const lead of ranked) {
  // toFixed(0) 只是讓輸出好讀
  console.log(`${lead.company}：期望值 ${lead.expectedValue.toFixed(0)}`);
}

// 輸出：
// B 公司：期望值 48
// C 公司：期望值 38   (原文為 37.5，toFixed 四捨五入成 38)
// A 公司：期望值 18
```

注意排序結果：最「理想」的 A 公司因為錄取機率太低，期望值反而墊底；把握度高的 B 公司排到最前面。這正是期望值排序想提醒你的事——**別只看工作有多好，也要看你拿到的機率有多高**。實務上你用一份試算表就能完成同樣的計算，不需要真的寫程式。

## 常見陷阱

!!! warning "被求職流程「吸進去」，浪費時間在你根本不會去的公司"
    求職流程一旦啟動就很容易讓人陷進去：投了申請、對方回信、約了面試、開始準備……等你回過神，已經投入好幾天，卻是在一份「就算錄取也不會接受」的工作上。**先篩選、再投入**，逼自己在花力氣之前回答「拿到 offer 我真的會去嗎？」。答案是「不會」的，提前劃掉。

!!! warning "把評分系統做得太複雜，結果遲遲無法行動"
    篩選的重點是幫你「快速排序」，不是打造一套完美的量化模型。權重與欄位越簡單越好；一旦系統複雜到讓你不想維護、不敢下手，它就失去了意義。**Simpler is better。**

!!! warning "誤以為大公司就等於工作穩定"
    大企業一樣會經歷重組（restructuring），也常常沒什麼明顯理由就裁員。評估 job security 時別只看公司規模，把「這份工作的實際保障風險」當成一個獨立的加分條件來衡量。

!!! warning "低估「培訓＋派遣」型合約的隱藏成本"
    這類方案常有低培訓時薪、高淘汰率、兩年綁約與高額違約金，而且培訓與待派空窗期可能不計入合約年限，實際把你綁得更久。簽約前務必查 Glassdoor／Reddit 口碑、讀完每一條條款、算清總財務義務，並確認自己有存款撐過空窗期。

## 練習

1. 打開你的職缺試算表，逐一檢視每個 lead，**把不符合任何「必備條件（must-have）」的直接劃掉**。
2. 對剩下的職缺，用你的「加分條件（nice-to-have）」給每份工作打分、加總，排出偏好順序。權重與計分方式盡量簡單。
3. 新增一個欄位：**用合理努力拿到這份工作的機率（百分比）**。把它乘上工作的價值分數，得到 **expected value（期望值）**，再依期望值重新排序，找出最值得你花時間的職缺。
4. 閱讀 The Odin Project 的 [Hire Beware 部落格文章](https://github.com/TheOdinProject/blog/blob/main/hire-beware.md)，了解「培訓＋派遣」型機會的警訊。文章要點已整理在本課「特別小心『培訓＋派遣』型的機會」一節；建議搭配原文一起看，並把其中的檢查項目（查口碑、讀合約、算財務義務、備妥存款）納入你評估這類職缺的流程。
