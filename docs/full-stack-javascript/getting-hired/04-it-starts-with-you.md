# 從你自己開始

> 改寫自 The Odin Project：[It Starts with YOU](https://www.theodinproject.com/lessons/node-path-getting-hired-it-starts-with-you)
> ｜Full Stack JavaScript › Getting Hired › Preparing for Your Job Search

## 核心概念

找工作很容易變成一件「向外看」的事：刷求職網站、比較職缺、投出一份又一份履歷，然後被動地等回音。但這一課要你先做一件相反的事——**向內看**。如果你不了解自己，就算真的拿到 offer（錄取通知），也很可能對最後落腳的地方感到不滿意。花點時間做自我評估（self-assessment）與目標設定（goal-setting），能把「找工作」這個混亂、令人焦慮的過程，變成一個更好掌握、更有策略的過程。

這一課沒有程式碼，卻是整個求職單元的地基。它要求你誠實回答四組問題：**我現在的處境是什麼？我想去哪裡？哪些條件是不可妥協的？我手上有什麼籌碼、又有哪些弱點？** 把這四件事想清楚，後面挑職缺、寫履歷、面試講故事，全都會變得更聚焦。

### 你現在在哪裡（你的「需要」）

在定義出「自己真正需要什麼」之前，你根本沒有標準去判斷哪條路才是對的路。所以先靜下來，把當下（對一份工作而言）最關鍵的需求想清楚。下面這些問題，能幫你分辨哪些是**真正必要**的，哪些只是**有更好**的：

- 我為什麼「需要」一份工作？
- 我需要什麼樣的薪資水準，才能養活自己與家人？
- 我需要什麼程度的工作穩定性（job security），才能支撐自己與家人的生活？
- 我「現在」有多急著要拿到一份工作？

這一組問題的關鍵字是「需要（need）」，不是「想要（want）」。它逼你面對現實：如果你下個月就得付房租，那你的處境和一個有半年存款、可以慢慢挑的人完全不同，後續的策略也應該不同。

### 你想去哪裡（你的「想要」）

對自己的需要誠實之後，接著才輪到「想要」。你未來的需求會是什麼？你想在什麼樣的公司工作？這一層是關於長期方向與偏好，可以問自己：

- 我為什麼「想要」投入 web development（網頁開發）這個領域？
- 我想要什麼樣的職涯路徑（career path）？
- 我想待在一間由開發者主導（developer-led）的公司，還是由業務或商業導向（sales- or business-led）主導的公司？
- 我偏好團隊協作的環境，還是遠端（remotely）工作？
- 我喜歡長工時、假日也加班，還是朝九晚六規律上下班？
- 「工作內容」對我來說比「一起工作的人」更重要嗎？
- 我一定要在乎自己所打造的產品嗎？
- 我一定要在乎自己所使用的技術嗎？
- 我偏好「輕鬆」的工作環境，還是一個會不斷鞭策我成長的環境？
- 我想不想要免費食物、啤酒這類「生活小確幸」的福利？
- 我一定要享受我的工作，還是就算是爛工作我也能硬撐過去？

這些問題沒有標準答案，重點是誠實。有人願意為了學習速度忍受高壓；有人寧可薪水少一點也要遠端；有人在乎產品的意義，有人只在乎技術是否有趣。沒有對錯，只有「這是不是你」。

### 把需要與想要排出優先順序

回答完上面兩組問題後，你手上至少會有一份非正式的清單，列著對你重要的事。現在，**狠下心來替它們排序**。哪些是你的「不可妥協（non-negotiables）」，哪些只是「有更好（nice-to-haves）」？

每一間公司、每一個機會，都會落在這些問題所定義出的光譜（spectrum）上某個位置——而這些問題，正是你之後用來評估它們的標準。先排好順序，你在面對機會時才不會亂了陣腳，也不會為了追逐一些「其實你根本不會選」的東西而把自己攤得太薄。

具體做法：**確認你知道每一項「必須有（must-haves）」，以及「有更好」項目裡最重要的前五名。** 這份排序就是你之後篩選職缺的過濾器。

### 你的資產與負債是什麼

該對自己誠實了——**你能為對方帶來什麼？** 如果你正處於職涯早期（你多半是），你得下功夫讓自己從一大票「不夠格的候選人」中脫穎而出。這裡要拿捏一條界線：既要對自己誠實，又不能把自己賣得太廉價（sell yourself short）。

每個人、尤其是剛起步的人，都會覺得自己不夠格。這很正常，別為此煩惱。這個練習不是要你自我否定，而是要**找出你的強項與弱項**，好讓你能量身打造一套故事：突顯強項，並誠實面對弱項。

一些可以問自己的問題（你在早期面試裡很可能會再聽到一遍）：

- 我有多「飢渴（hungry）」、多想要成長？
- 我寫後端（back end）程式的能力如何？
- 我組前端（front end）的能力如何？
- 我能解決很技術性的問題嗎，例如腦筋急轉彎和演算法（algorithms）？
- 我做過能展現多元技能的專案（projects）嗎？
- 我認識能替我背書、推薦（recommend）我的人嗎？
- 我擅長與人相處、容易建立關係嗎？
- 我散發出的是自信的氣場，還是顯得害羞？
- 我是不是常常低估自己與自己的能力？
- 我有沒有廣泛的一般技術知識（general tech knowledge）？
- 我對某個特定的技術領域或產業垂直領域（vertical）特別熟悉嗎？（轉職者通常都有一個）
- 我傾向拖延，還是能把事情做完（get things done）？
- 我會乾等別人來幫忙，還是主動去把答案找出來？
- 我是迎向挑戰，還是逃避挑戰？
- 我以前解決過哪些真正困難的問題？

在經驗不多的情況下投履歷，意味著你必須**補強弱點、放大強項**。對幾乎所有人來說，你最大的強項會是「飢渴」與「學習能力」。但公司早就聽膩這套說法了，所以你得再綁上其他強項，把你的故事說得有說服力。

### 飢渴（hunger）不等於絕望（desperation）

最後一個重要區分：**飢渴不等於絕望。** 飢渴是「追求獎賞」——你可以在一個處境還算舒適、正在優化機會的狀態下感到飢渴；絕望則是「逃避失敗」——那是當你「非要這份工作不可」時才會出現的狀態。

無論你內心其實有多絕望，都要竭盡所能不讓自己「聽起來很絕望」。原因很現實：面試官能嗅到絕望，而絕望會削弱你的談判籌碼，也讓對方懷疑你是不是「隨便一份工作都好」，而不是真的適合這個團隊。把焦點放在「我想追求什麼」，而不是「我不能失去什麼」。

## 程式碼範例

本課以觀念為主，沒有需要執行的程式。這裡把它濃縮成一份可以照著填的自我評估清單，你可以複製到自己的筆記軟體裡逐項作答：

```text
# 自我評估清單（Self-Assessment）

## 1. 我現在在哪裡（需要 / needs）
- 我為什麼需要一份工作？
- 我需要的最低薪資：____
- 我需要的工作穩定度：____
- 我現在有多急（1-10）：____

## 2. 我想去哪裡（想要 / wants）
- 我想要的職涯方向：____
- 公司文化偏好（團隊/遠端、工時、產品）：____
- 我在乎的技術 / 產品：____

## 3. 優先順序（Ranking）
- 不可妥協（must-haves）：____
- 有更好 Top 5（nice-to-haves）：
  1. ____  2. ____  3. ____  4. ____  5. ____

## 4. 資產與負債（Assets / Liabilities）
- 我最強的 3 個強項：____
- 我最需要補強的 3 個弱項：____
- 我解過最難的問題：____
- 能替我背書的人：____
```

## 常見陷阱

!!! warning "跳過自我評估，直接海投履歷"
    最常見的錯誤，就是覺得「這一課沒有程式碼、不重要」，於是略過不做，直接開始大量投遞。結果就是四處亂投、每一份都不夠貼合，也沒有一套清楚的標準去判斷哪個 offer 真正適合你。先花一兩個小時把這四組問題想清楚，之後篩選職缺、寫求職信、面試講故事的效率都會大幅提升。

!!! warning "把「飢渴」演成「絕望」"
    在履歷與面試中傳達出「我什麼工作都願意做、拜託給我機會」的訊號，會讓你顯得絕望。絕望會削弱談判籌碼，也讓對方懷疑你不是真的適合。永遠把重點放在「我想追求什麼、能帶來什麼」，而不是「我不能失去什麼」。

!!! warning "為了謙虛而低估自己"
    覺得自己不夠格是每個新手的正常心理，但別讓它變成在面試中不斷貶低自己。這個練習的目的是誠實盤點強弱項，好讓你「突顯強項、坦然面對弱項」，不是要你把自己賣得太廉價。

## 練習

這一課的 Assignment（作業）就是完成一次完整的自我評估。建議按下面的步驟做，並把答案寫下來（寫下來才算數，光在腦中想很容易自欺）：

1. **盤點需要**：回答「你現在在哪裡」那組問題，寫出你對薪資、穩定度與急迫程度的真實需求。→ 驗證：你能用一句話說出「我現在最低限度需要的是什麼」。
2. **描繪想要**：回答「你想去哪裡」那組問題，寫出你偏好的公司類型、工作方式與在乎的事。→ 驗證：清單裡至少有 5 項具體偏好。
3. **狠心排序**：把上面所有項目分成「不可妥協」與「有更好」，並替「有更好」排出前五名。→ 驗證：你能明確指出每一項 must-have，以及 nice-to-have 的 Top 5。
4. **盤點資產與負債**：回答「你的資產與負債」那組問題，列出你的強項、弱項、解過的難題與可背書的人脈。→ 驗證：你能講出一個「以強項為主軸、誠實帶到弱項」的個人故事雛形。

做完後把這份清單存好，之後每一課（履歷、作品集、面試）都會回頭用到它。
