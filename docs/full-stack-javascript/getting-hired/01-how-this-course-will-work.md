# 這門課怎麼進行

> 改寫自 The Odin Project：[How This Course Will Work](https://www.theodinproject.com/lessons/node-path-getting-hired-how-this-course-will-work)
> ｜Full Stack JavaScript › Getting Hired › Preparing for Your Job Search

## 核心概念

你已經花了很長時間鑽研 web development（網頁開發），從零開始寫出一個又一個 app（應用程式）。現在，終於到了讓別人付錢請你繼續邊做邊學的階段——也就是找到第一份正職工作。這個「Getting Hired（找到工作）」課程和你之前上過的所有課程都不一樣，它的重點不再是「學會某項技術」或「做出某個專案」，而是把你這個人推向就業市場，用有系統的方法把你送進一家公司。

### 這是一場機率遊戲，但機率能被你自己改變

先講一個很重要、但很多人不願意面對的現實：**找工作，本質上是一場機率遊戲（odds game）**。對還沒有正式工作經驗、還沒被過往履歷「證明」過的第一份工作求職者來說更是如此。你不是「能力不夠所以找不到」，而是「就算能力足夠，也需要投出夠多次、被夠多人看見，才會命中」。

這代表兩件事：

1. **你一定會被拒絕，而且不只一次。** 每一個工程師在職涯早期都吃過大量的拒絕信、已讀不回、面試後石沉大海。這不是你的失敗，這是這場遊戲的一部分。把每一次 rejection（拒絕）當成統計上的必然，而不是對你個人價值的否定。
2. **你能做的是「提高命中率」與「增加投擲次數」。** 一邊持續強化技能、把 portfolio（作品集）做得更亮眼，讓自己成為更強的候選人；一邊每天持續投遞、持續讓自己曝光。技能決定你「每次被看見時的命中率」，投遞量決定你「總共有幾次機會」。兩者相乘，才是你真正的錄取機率。

很多人會犯的錯，是在其中一邊過度投入、另一邊完全停擺。有人躲起來狂刷技術、把作品集打磨到完美，卻遲遲不敢投履歷；也有人海投上百份、卻從不回頭補強自己的弱點。這門課要教你的，是「兩件事同時做」——持續變強，也持續出手。

### 為什麼需要一條結構化的路徑

找工作最危險的做法，是「沒有計畫就一頭栽進去」。你會很容易陷入焦慮的亂槍打鳥：今天投十家、明天改履歷改到半夜、後天又聽某篇文章說要學某個新框架……到頭來每件事都做了一點，卻沒有一件做到位，還把自己搞得精疲力盡。

這門課存在的意義，就是給你一個**有結構的路徑（structured path）**，像鷹架（scaffold）一樣把「找工作」這個混亂的過程一步一步撐起來，讓你避開常見的陷阱。你不需要自己憑空發明流程，只要跟著課程的順序走，每一步都有明確的目標與可檢核的產出。

課程作者給了一個關鍵的使用方式，值得逐字記住：

> **先把整個課程完整讀過一遍，然後在實際求職的過程中，再一步一步照著它走。**

先通讀一遍，是為了讓你對「接下來會發生什麼」有全局的概念——知道要準備履歷、要經營作品集、要練習面試、要追蹤投遞紀錄等等；等到真的開始行動時，再把課程當成一份逐步操作的 checklist（檢查清單），一課一課落實。這種「先鳥瞰、再落地」的兩段式讀法，會比你邊讀邊慌著執行有效率得多。

### 對自己誠實：你想要什麼、你能提供什麼

課程一開始就提醒你一件事：要對自己**誠實**。這裡的誠實有兩個方向：

- **你想要什麼（what you want）：** 你想進大公司還是小新創？想遠端還是進辦公室？前端、後端還是全端？你能接受的最低薪資、能承受的通勤距離、想要的團隊文化是什麼？把這些想清楚，才不會在收到 offer（錄取通知）時手足無措，也才不會浪費時間去追一個你其實根本不想要的職位。
- **你能提供什麼（what you can offer）：** 誠實評估自己目前的技術水準、作品集的完成度、可以拿出來說嘴的專案。這不是要你妄自菲薄，而是要你知道自己現在站在哪裡，才知道還要補強哪裡，也才能在面試中不誇大、不心虛地介紹自己。

把你走到這一步所用的那份「嚴謹與紀律（rigor and structure）」——也就是你能一路啃完前面所有課程的那股韌性——原封不動地搬到找工作這件事上。你已經證明過自己能靠結構化的努力學會很難的東西了，找工作也是同一回事。

### 這門課的格式與每一課的長相

因為重點是「把你推出去」而不是「教你一項技術」，這門課的編排會和前面的技術課明顯不同。課程中確實會有**少數幾個專案（projects）**要你動手做，但大部分的內容都是為了替你的求職流程搭建鷹架而存在。

每一課大致會照著同一個節奏走，理解這個節奏能讓你上起來更順：

1. **介紹主題並說明它在整體求職中的定位。** 先讓你知道「這一課在整張地圖上的哪個位置、為什麼此刻要學它」。
2. **請你去讀文章、看影片、上線上課程，或以其他方式吸收內容。** 課程會大量引用外部優質資源來補足細節。
3. **常常會請你完成具體的任務（tasks）。** 例如要你記錄自己投遞過哪些職缺。這些任務純粹是為了幫助你在流程中不迷路、不遺漏，而不是為了考核你。
4. **最後在每一課結尾附上額外的延伸資源與可能有用的小提醒（tidbits）。**

當你認出這個固定結構後，就能用同一套方式面對每一課：先看它在地圖上的位置，再吸收指定內容，接著確實完成任務、留下紀錄，最後掃過延伸資源。

### 心態：把找工作當成一個每天推進的專案

最後，把整件事的心態校正好。找工作不是一次性的衝刺，而是一段需要每天推進的過程。你要做的，就是**每天都讓自己往前一點**——多投一份履歷、多修一段作品集的說明、多練一題面試考古題、多認識一個業界的人。技能與作品集讓你「更值得被錄取」，持續的曝光與投遞讓你「有更多被錄取的機會」。兩者一起做，時間拉長，錄取只是機率累積後的必然結果。

會被拒絕，別人也一樣會；差別只在於，有計畫的人會把每一次拒絕轉化成下一次更好的出手。跟著這門課的結構走，然後——用課程作者的話說——放手去把工作拿下吧（go get hired）。

## 程式碼範例

本課以觀念與求職流程為主，沒有需要執行的程式碼。這裡改用一份「行動節奏」的操作清單，幫你把上面的觀念落成每天可執行的動作：

```text
求職這門課的使用節奏（先鳥瞰、再落地）

第 0 步：先把整個 Getting Hired 課程「完整讀過一遍」，建立全局概念
第 1 步：對自己誠實地寫下「我想要什麼」與「我能提供什麼」
第 2 步：進入每一課時，照固定節奏走：
        看定位 → 吸收指定內容 → 完成任務並留下紀錄 → 掃延伸資源
第 3 步：把「持續變強」與「持續出手」排進每天的行程：
        - 變強：修作品集 / 練面試題 / 補技術弱點
        - 出手：投遞履歷 / 追蹤投遞狀態 / 拓展人脈
第 4 步：把每一次 rejection 記錄下來，當成統計數字，而非自我否定
```

## 常見陷阱

!!! warning "只顧「變強」卻遲遲不「出手」"
    很多人把找工作誤解成「先把技術與作品集練到完美，再開始投履歷」。但這是一場機率遊戲——你投出去的次數本身就是變數之一。躲起來無限打磨、遲遲不敢出手，等於自願放棄一大半的機會。正確做法是「持續變強」與「持續投遞」同時進行，不要等到覺得自己準備好了才開始，因為那一天不會自己到來。

!!! warning "把拒絕當成對自己價值的否定"
    第一份工作的求職者一定會收到大量拒絕，這是機率遊戲的必然，不是你能力不足的證明。若把每一次 rejection 都內化成「我不夠好」，你會越投越沒信心，最後乾脆停手。請把拒絕當成統計數字看待，並從中找出「下一次能改進的一件事」，然後繼續出手。

!!! warning "沒有計畫就一頭栽進去"
    在毫無結構的情況下亂投履歷、隨手改履歷、聽到什麼就學什麼，會讓你每件事都做一半、還把自己累垮。這門課的價值就是提供一條結構化路徑——請務必先完整讀過一遍再開始，並在過程中把它當成逐步操作的清單，而不是憑感覺行動。

## 練習

課程本身在這一課沒有指定要交付的作業，但作者給了一個明確且重要的指示，請照做：

1. **完整讀過整個 Getting Hired 課程一遍。** 不用馬上執行每一課的任務，目標是先在腦中建立「接下來會經歷哪些階段」的全局地圖，知道履歷、作品集、面試準備、投遞追蹤等環節各自落在哪裡。
2. **回到起點，開始逐課落地。** 通讀之後，再從第一課開始，按照課程順序一步一步實作，把每一課裡「請你完成的任務」確實做完並留下紀錄（例如投遞紀錄表）。
3. **把「變強」與「出手」排進每天的行程。** 每天至少各推進一項：一邊補強技能與作品集，一邊持續投遞與拓展人脈。
4. **建立你自己的求職紀錄。** 從現在起，準備一個地方（試算表或筆記皆可）記錄你投過哪些公司、進度到哪、收到什麼回覆，後續課程會需要用到這份紀錄。

若某些步驟涉及特定作業系統或特定專案的操作細節，請以原文課程的指示為準。
