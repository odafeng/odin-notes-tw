# 4. AI 並行生成講義內容 + 事後校正

日期：2026-07-04

## 狀態

已接受

## 背景

要把 8 門課、198 課消化改寫成自足繁中講義。逐課手寫耗時極大；憲法要求
「改寫不翻譯、自足、禁簡體、技術詞附中文」。需要在可控品質下大量產出。

## 決策

用**多 agent 並行 workflow** 生成：每課一個 agent，讀 vendor 原始教材 +
WebFetch 指定閱讀，依統一模板改寫，寫回 `docs/.../NN-slug.md`。因伺服器
限流，改為**每批 3 個節流**。產出後跑**機械式 QC**（無簡體、章節齊、footer、
無「學習目標」段），再跑一輪**外科式內容校正 workflow**（逐課對照原文修技術/
程式碼/術語錯誤）。每段完成即 `mkdocs build --strict` + commit + 部署。

## 後果

- 好處：短時間產出全部 198 課並上線；流程可重跑、可逐課補產、可逐課校正。
- 代價／取捨：
  - **AI 生成內容必然有錯**（技術謬誤、程式碼未實跑、幻覺）；機械式 QC 抓不到語意錯，
    需人工抽查 + 校正 workflow 降低風險，但**無法保證逐課正確**。
  - 一次高並發會觸發 rate limit；內容過濾偶會誤擋 agent 回傳（檔案其實已寫好）。
  - 不同 agent 可能術語不一致，需事後統一。
  - 大量 token 成本。
